# Spaceship Titanic — ML Project

Predicts which passengers were transported to an alternate dimension.
Built following [PROJECT_RULES.md](PROJECT_RULES.md) architecture.

---

## Project Structure

```
project/
├── configs/baseline.yaml        # All experiment parameters. Edit here, not in code.
├── data/
│   ├── raw/                     # Original Kaggle files. Immutable.
│   ├── interim/                 # Structurally processed splits.
│   └── processed/               # Pipeline-ready arrays for the model.
├── reports/figures/             # Evaluation plots.
├── mlruns/                      # Auto-generated by MLflow. Never touch.
└── src/
    ├── utils.py                 # Shared logic (NULL_STRINGS, replace_string_nulls)
    ├── data/make_dataset.py     # Null-string detection + structural split
    ├── features/build_features.py  # Full sklearn preprocessing pipeline
    ├── models/train_model.py    # XGBoost training + MLflow logging
    ├── models/predict_model.py  # Generates submission.csv
    └── visualization/plots.py  # Reusable plot functions
```

---

## What Each Script Does

| Script | Reads from | Writes to | Responsibility |
|--------|-----------|-----------|----------------|
| `src/data/make_dataset.py` | `data/raw/` | `data/interim/` | Null-string detection for threshold decisions + train/val/test split |
| `src/features/build_features.py` | `data/interim/` | `data/processed/` | Full sklearn pipeline: null replacement → cabin decomposition → feature engineering → impute/encode/scale → optional RFE |
| `src/models/train_model.py` | `data/processed/` | `mlruns/`, `reports/figures/` | XGBoost + early stopping + MLflow tracking |
| `src/models/predict_model.py` | `mlruns/`, `data/interim/` | `data/raw/submission.csv` | Loads full pipeline from MLflow, generates Kaggle submission |
| `src/utils.py` | — | — | Single source of truth for null-like string detection, shared by make_dataset and the pipeline |

---

## What the Sklearn Pipeline Does (in order)

1. **NullStringReplacer** — converts `"None"`, `"null"`, `""`, `"unknown"`, etc. → `np.nan`
2. **DropColumnsTransformer** — removes `Name`
3. **GroupIdTransformer** — extracts `group_id` from `PassengerId`, drops `PassengerId`
4. **CabinTransformer** — decomposes `Cabin` into `cabin_deck`, `cabin_num`, `cabin_side`, drops `Cabin`
5. **TotalSpendTransformer** — engineers `total_spend` = sum of all luxury service columns
6. **ColumnTransformer** — imputation + encoding + scaling per column type

The full pipeline (`preprocessor → optional RFE → model`) is what gets saved to MLflow. At inference, pass raw data exactly as the source sends it — the pipeline handles everything.

## What You Can Change in `configs/baseline.yaml`

```yaml
# Null thresholds for structural column/row dropping
null_handling:
  drop_column_threshold: 0.60   # Drop column if null ratio > 60%
  drop_row_threshold:    0.50   # Drop row from train if null ratio > 50%

# Categorical encoding
features:
  encoding: 'ordinal'           # Options: 'ordinal' | 'onehot' | 'target'
  numerical_imputer: 'median'   # Options: 'median' | 'mean' | 'constant'
  categorical_imputer: 'most_frequent'  # Options: 'most_frequent' | 'constant'

# RFE feature selection — toggle without touching code
rfe:
  enabled: false                # Set true to activate
  n_features_to_select: 10
  step: 1

# XGBoost hyperparameters
model:
  params:
    n_estimators: 500
    max_depth: 6
    learning_rate: 0.05
    subsample: 0.8
    colsample_bytree: 0.8
    ...
```

## How to Run a Full Experiment

### Step 0 — Setup (once)

```bash
uv sync
uv pip install -e .
```

### Step 1 — Structural processing + split

```bash
python -m src.data.make_dataset
```

Output: `data/interim/train_split.parquet`, `val_split.parquet`, `test_original.parquet`

### Step 2 — Build preprocessing pipeline

```bash
python -m src.features.build_features
```

Output: `data/processed/X_train.parquet`, `X_val.parquet`, `X_test.parquet`, `y_train.parquet`, `y_val.parquet`, `preprocessor.pkl`

### Step 3 — Train + log to MLflow

```bash
python -m src.models.train_model
```

Output: run logged to `mlruns/` with parameters, metrics, plots, and the full production pipeline as artifact.

### Step 4 — View results in MLflow UI

```bash
mlflow ui
```

Open your browser at **http://localhost:5000**

You will see:
- All runs grouped under the experiment name
- Parameters logged from `configs/baseline.yaml`
- Metrics: `accuracy`, `roc_auc`, `f1`, `precision`, `recall`, `log_loss`, `avg_precision`
- Artifacts tab: confusion matrix, ROC curve, precision-recall curve, feature importance, training history plots
- The full production pipeline saved as `production_pipeline/`

Copy the **Run ID** from the UI (shown under the run name) — you need it for the next step.

### Step 5 — Generate Kaggle submission

```bash
python -m src.models.predict_model <RUN_ID>
```

Replace `<RUN_ID>` with the ID copied from MLflow UI. Example:

```bash
python -m src.models.predict_model 3f2a1b8c9d4e5f6a7b8c9d0e
```

Output: `data/raw/submission.csv` ready to upload to Kaggle.

## Running a New Experiment

1. Edit `configs/baseline.yaml` — change hyperparameters, encoding, or toggle RFE.
2. If you changed features or encoding → re-run steps 2 and 3.
3. If you only changed model hyperparameters → re-run step 3 only.
4. Compare runs side by side in `mlflow ui`.

## Key Design Rules

- Every value transformation lives inside the sklearn pipeline — no pandas transformations outside of it.
- The pipeline is `fit` only on `train_split`. Val and test always use `transform` only.
- `group_size` is not a feature — it requires full dataset context and cannot be computed at inference for a single row.
- `NULL_STRINGS` is defined once in `src/utils.py` and shared by both `make_dataset.py` and `NullStringReplacer` in the pipeline.
